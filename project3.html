Project 3: Data Collection Application

Introduction

After I finished my Interact Club website as detailed in Project 2, I realized the user could not directly edit the Google Docs sign up sheet from the website. After finding this limitation, I learned how to collect, manipulate, and store data directly from the website using JavaScript and jQuery. This led me to my second implementation you can access below by going to the custom Interact Club website, going to the the Event Sign Ups page, and scrolling down the section titled "Built-In Sign Up Implementation:"


Description

To start, I went back to the discovery stage and learned how to collect, change, and store information from the user through a website. I was specifcally taught how to save this information to my computer's local storage and a built-in cloud only accessible to me from the learning website. For the project itself, I decided to collect data using built-in Google prompt commands for ease of use and save this data to local storage where it would be remain intact on the device if the user refreshed or deleted the page.

You may be wondering why I decided to use local storage instead of cloud storage since local storage doesn't save data to other computers for a sign up. Well, this implementation is meant to show the capabilities of the website and give a basic framework for understanding how data can be collected, altered, and stored. Cloud storage is a much harder concept to implement and requires access to potientally expensive cloud services and APIs compared to local storage which I can easily access through JavaScript commands. In the future, I can simply edit the preexisting code to work with cloud storage if desired but for now, local storage is an adequate demonstration I was able to implement in the short time left in my course.

Overall, I am satisfied with the ability of my page to collect, manipulate, and store user data to emulate a sign up for the Interact Club website. Although it cannot connect to the cloud, it serves its purpose to demonstrate how a built-in sign upfor a website would work compared to a third party service like Google Docs. I am now confident in my abilities to collect, store, and manipulate data and am excited to learn more about this topic and more website design aspects in the future.
